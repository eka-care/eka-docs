## About this endpoint

`GET /med-link/registry/` returns every ontology available for codification,
the versions of each ontology, and the active indexes backing them.

Call it before linking to confirm the exact `ontology` and `version` values to
send to [Link Entity](/api-reference/medical-entity-codification/link-entity) and
[Link Entities (Batch)](/api-reference/medical-entity-codification/link-entity-batch).

## Response fields

| Field | Description |
|-------|-------------|
| `ontologies[]` | Available ontologies, each with its `name` and supported `versions`. |
| `indexes[]` | The active indexes serving each ontology/version pair. |
| `indexes[].status` | Index status — `ready` indexes can serve requests. |
| `indexes[].is_active` | Whether the index is currently active for linking. |

<Tip>
Use the `ontology` + `version` pairs from `ontologies[]` directly as the
`ontology` and `version` fields in linking requests.
</Tip>
