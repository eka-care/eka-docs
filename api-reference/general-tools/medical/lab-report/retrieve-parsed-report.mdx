---
title: "Retrieve Parsed Report"
---

Use the `https://api.eka.care/health/api/v1/fhir/retrieve?identifier={identifier}` endpoint along with the identifier which you have recieved in webhook events to fetch results. This endpoint returns a FHIR response.

- **Authorizations (string | header | required)**
    
    Bearer authentication header of the form **`Bearer <token>`**, where **`<token>`** is your auth access token.
    
- **cURL/API to retrieve parsed data:**
```bash
curl --request GET \
  --url --url 'https://api.eka.care/health/api/v1/fhir/retrieve?identifier=unique_identifier' \
  --header 'Authorization: Bearer <token>'
```
    
    - **Query Parameters**
        - **identifier (string> | required)**
            A unique identifier for the uploaded medical record recieved in webhook
