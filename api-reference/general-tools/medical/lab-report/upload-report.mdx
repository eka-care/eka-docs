Upload the image or PDF file following the document specifications below. Ensure you define the document type and tasks you want to process in the query parameters.
**Document Specifications**
- **PDF Documents**: Ensure the file is complete and legible.
- **Images**:
    - Preferred DPI: **150+**
    - Images must be **non-blurred**
    - **Avoid** images with a high degree of **non-linear warping** or **rotation** for optimal processing accuracy.

**cURL/API to upload medical record**
```bash
curl --location 'https://api.eka.care/mr/api/v2/docs' \
--header 'Accept: */*' \
--header 'User-Agent: YourAppName' \
--header 'auth: <YOUR_ACCESS_TOKEN>' \
--form 'file=@"/path/to/your/file.pdf"' \
--form 'dt=lr' \
--form 'task=smart' \
--form 'task=pii' \
--form 'task=classification'
```

- **Authorizations (string | header | required)**
    - Bearer authentication header of the form **`Bearer <token>`**, where **`<token>`** is your auth access token.
- **Query Parameters**
    - **task (required)**
        
        Specify the tasks you want to perform. 
        
        - `smart`: Smart
        - `pii`: PII
        - `classification`: Document Classification
    - **dt** (enum \<string\> | required)
        Specifies the type of document being uploaded. The following values are allowed:
        - **`ps`**: Prescription
        - **`lr`**: Lab Report
        - **`ot`**: Other
        - **`ds`**: Discharge Summary
        - **`vc`**: Vaccine Certificate
        - **`in`**: Insurance
        - **`iv`**: Invoice
        - **`sc`**: Scan
    - **file (required)**
        
        Upload the image or PDF file you want to process.
        
- **Response**
**document_id (string)**
The unique identifier for the uploaded document.
