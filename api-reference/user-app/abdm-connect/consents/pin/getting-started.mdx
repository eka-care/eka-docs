---
title: Overview
---


To Approve/Revoke consent requests, the user must first set a consent PIN.
The user can check if a consent PIN exists for him/her, create a new PIN, or change an existing PIN.

<Note> User Authorization will be required for all these operations.</Note>

<Steps>
    <Step title="Check Existing PIN: Exist PIN">
        Use the [Exist PIN](/api-reference/user-app/abdm-connect/consents/pin/exists) endpoint to check if the user
        has already set a consent PIN.
    </Step>
    <Step title="Create New PIN: Set PIN">
        Use the [Set PIN](/api-reference/user-app/abdm-connect/consents/pin/set) endpoint to create a new consent PIN.
    </Step>
    <Step title="Update Existing PIN: Change PIN">
        Use the [Change PIN](/api-reference/user-app/abdm-connect/consents/pin/change) endpoint to change the
        existing consent PIN. It will need the old PIN and the new PIN.
    </Step>
</Steps>
